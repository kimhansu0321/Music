# 🎧 한국 음악 추천 및 공유 플랫폼 (K-Music Share)

## 📌 프로젝트 개요
사용자들이 자신이 좋아하는 우리나라(한국) 음악을 소개하고, 다른 이용자들과 다양한 장르의 음악 추천을 공유할 수 있는 커뮤니티 기반 웹 프론트엔드/백엔드 플랫폼입니다. K-Pop, 인디 밴드, 힙합, 발라드, 트로트 등 다양한 한국 음악을 알리고 듣는 즐거움을 나눌 수 있습니다.

---

## ✨ 핵심 기능 (Core Features)

### 1. 음악 추천 게시판 (Feed)
* **음악 정보 등록:** 곡 제목, 아티스트 이름, 앨범 커버 이미지, 추천 이유(감상평) 입력.
* **미디어 링크 첨부:** 유튜브(YouTube) 재생 링크나 스포티파이, 멜론 등의 링크를 첨부하여 바로 들을 수 있게 지원.
* **해시태그 및 장르 태그 기능:** #KPOP, #인디, #여름밤에듣기좋은노래 등 태그를 달아 검색 최적화.

### 2. 필터링 및 검색 (Search & Filter)
* **장르 및 테마별 큐레이션:** '비 오는 날 듣기 좋은 곡', '드라이브 할 때 좋은 곡' 등으로 정렬.
* 검색창을 통해 아티스트명이나 곡 제목으로 기존 추천글 탐색.

### 3. 소셜 상호작용 (Social Interaction)
* **좋아요(Like) 및 댓글(Comment):** 다른 사용자의 추천글에 공감하거나 감상평 댓글 달기.
* **이달의 베스트 추천:** 가장 많은 추천(좋아요)을 받은 게시물을 홈페이지 상단에 하이라이트 노출.

### 4. 사용자 계정 (User Account) - 확장 기능
* 회원가입/로그인 (간편 로그인 연동).
* '내 플레이리스트(My Playlist)': 내가 추천한 곡들과 좋아요를 누른 곡들을 따로 모아보는 기능.

## 🛠 기술 스택 추천 (Tech Stack)

### 프론트엔드 (Frontend)
* **HTML5 / CSS3 / JavaScript:** 기본적이고 직관적인 웹 개발.
* **최신 프레임워크 (선택):** React.js 혹은 Vue.js를 사용하면 '좋아요'나 '댓글' 실시간 반영 시 매우 부드러운 UI를 만들 수 있습니다.
* **디자인 CSS 참고:** Tailwind CSS, Bootstrap 활용하여 빠르고 깔끔한 모바일-웹 반응형 디자인 구현.

### 백엔드 및 데이터베이스 (Backend & DB)
* **초기 프로토타입:** 브라우저의 `LocalStorage`를 활용해 간단히 구현.
* **정식 서비스 적용:** 
  * **서버:** Node.js (Express)
  * **데이터베이스:** MySQL / PostgreSQL (관계형, 사용자 정보 및 게시글 관리) 또는 Firebase (실시간 DB 연동과 호스팅이 편리).

## 🚀 개발 및 작업 진행 방법 (Step-by-step)

### [Step 1] 기획 및 화면 설계 (UI/UX)
1. 메인 페이지: 추천된 음악 목록(Feed)이 카드 형태로 나열되는 직관적인 디자인 구성.
2. 글쓰기 폼(Form): 사용자가 음악 정보를 입력하는 창 디자인.
3. 상세 페이지: 클릭 시 음악 재생 링크와 추천 이유, 댓글이 보이는 공간 구성.
*(Figma 또는 손그림으로 간단한 와이어프레임을 그립니다.)*

### [Step 2] 정적 웹페이지 퍼블리싱 (Front-End)
1. `index.html`, `style.css`, `app.js` 파일 생성.
2. 음악 카드 리스트, 내비게이션 바, 글쓰기 버튼 등을 HTML과 CSS로 작성.
3. 반응형 웹(Responsive Web)으로 디자인하여 모바일 화면에서도 쾌적하게 보이도록 설정.

### [Step 3] 기본 기능 구현 (JavaScript + LocalStorage)
1. `app.js`에 사용자가 입력한 데이터(가수명, 제목, 추천사)를 가져와 배열(Array)로 저장하는 로직 작성.
2. 페이지를 새로고침해도 데이터가 날아가지 않도록 `localStorage.setItem()`, `getItem()`으로 데이터 보존 환경 조성.
3. 화면에 등록된 음악 리스트를 동적으로 렌더링.

### [Step 4] 백엔드 및 DB 도입 (Backend Integration)
1. Node.js 등 서버 환경 구축.
2. 사용자가 글을 작성하면 Database에 `INSERT`하고, 메인 화면 로딩 시 `SELECT`로 가져오는 API 생성.

### [Step 5] 서비스 다듬기 ও 배포
1. 유튜브 썸네일을 자동으로 가져와 표시하거나 예쁜 아이콘 추가.
2. Vercel, Netlify (프론트엔드) 및 Heroku, Render (백엔드) 등을 통해 무료로 웹 상에 배포.
3. 주변 친구들에게 링크를 공유하여 실제 음악을 추천받아 테스트 진행.
